# 醫療貸款送審集中化系統 Prototype

此專案為前端 MVP，對應 `PRD.md` 的四大模組：

- 案件建立（Wizard 三步驟）
- 文件集中歸檔（A/B/C/D 分類）
- 防呆檢核（進度條、必備文件鎖定送件）
- 看板追蹤（資料準備中 / 送交檢核中 / 要求補件 / 審核通過）

## 執行方式

1. 直接用瀏覽器開啟 `index.html`
2. 建議使用 Chrome / Edge 最新版

## 主要操作流程

1. 以營業單位角色建立案件
2. 於案件詳情上傳 A1、C1、D1 必備文件
3. 系統完整度達成後，送交風管
4. 切換風管角色進行「要求補件」或「審核通過」
5. 若退件，營業單位可查看退件原因並補件後重送

## Prototype 功能清單

- 角色切換：營業單位 / 風管徵審
- 案件看板：卡片化顯示與狀態欄位管理
- 儀表板統計：各狀態案件數
- 搜尋與篩選：企業名稱、案件編號、狀態
- 動態資料預覽：案件資料即時呈現
- 拖曳或按鈕上傳：文件歸類到指定項目
- 必備文件檢核：缺件時禁用送交按鈕
- 退件原因：由風管填寫，營業可追蹤
- 案件資料匯出：JSON 下載
- 本機儲存：使用 `localStorage`

## 資料說明

- `localStorage` key：`medicalLoanCases`
- 清除資料方式：瀏覽器開發者工具刪除該 key

- `index.html`：Prototype 單頁應用（HTML/CSS/JS）
